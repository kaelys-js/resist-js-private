# 02 — WAF: Coraza + OWASP CRS, Custom Rules, Managed Rulesets, Anomaly Scoring

## Context

The Web Application Firewall (WAF) is one of Cloudflare's most critical security features. It inspects every HTTP request against a set of rules and either blocks, challenges, logs, or allows based on match results. Cloudflare's WAF operates in the `http_request_firewall_custom` (custom rules) and `http_request_firewall_managed` (managed rulesets) phases of its Ruleset Engine.

Locally, we simulate the WAF using **Coraza** (`coraza-caddy`), the open-source WAF engine that is fully compatible with the OWASP Core Rule Set (CRS). Coraza runs as a Caddy middleware module, inspecting requests before they reach the upstream services.

The same Valibot schema drives both:
- **Local simulation** via Coraza SecRule directives (generated by `waf.ts`)
- **Production deployment** via Pulumi Cloudflare WAF Ruleset resources

### How Cloudflare WAF Works

1. A request arrives and passes through the WAF phases in order:
   - **Custom Rules** (`http_request_firewall_custom`): User-defined rules with wirefilter expressions
   - **Rate Limiting** (`http_ratelimit`): See 03-rate-limiting.md
   - **Managed Rules** (`http_request_firewall_managed`): Cloudflare Managed Ruleset, OWASP Core Ruleset
2. Each rule has an **expression** (wirefilter syntax) and an **action** (block, challenge, js_challenge, log, skip, managed_challenge).
3. Managed rulesets contain hundreds of pre-built rules grouped by category (SQLi, XSS, RCE, etc.).
4. The OWASP Core Ruleset uses **anomaly scoring** — each rule adds to a cumulative score, and the request is blocked only if the total exceeds a threshold (paranoia level).
5. WAF scores are exposed via `cf.waf.score`, `cf.waf.score.sqli`, `cf.waf.score.xss`, `cf.waf.score.rce` fields.

### Coraza WAF Engine

- Open-source, ModSecurity-compatible WAF engine written in Go.
- The `coraza-caddy` module integrates Coraza directly into Caddy's request pipeline.
- Natively loads OWASP CRS rule files (`.conf` format).
- Supports SecRule syntax for custom rules.
- Anomaly scoring mode: each rule increments a score; threshold-based blocking.
- Supports request body inspection, response body inspection, file upload scanning.
- Transaction-based: each request gets a fresh transaction with its own score accumulator.
- Configurable paranoia levels (1-4) controlling rule sensitivity.
- Supports `@rx` (regex), `@pm` (phrase match), `@ipMatch`, `@contains`, `@beginsWith`, `@endsWith` operators.

/**
 * Tokenize a CF wirefilter expression into components.
 *
 * @param expression - The wirefilter expression string.
 * @returns Result containing an array of token strings, or an error.
 */
export function tokenizeCfExpression(
	expression: v.InferOutput<typeof v.string>,
): Result<ReadonlyArray<string>> {
	const trimmed = expression.trim();

	if (trimmed.length === 0) {
		return err('WAF_EMPTY_EXPRESSION', 'Expression cannot be empty');
	}

	// Tokenize: split on whitespace but respect quoted strings and parentheses
	const tokens: Array<string> = [];
	let current = '';
	let inQuote = false;
	let quoteChar = '';
	let depth = 0;

	for (let i = 0; i < trimmed.length; i++) {
		const char = trimmed[i]!;

		if (inQuote) {
			current += char;
			if (char === quoteChar) {
				inQuote = false;
			}
			continue;
		}

		if (char === '"' || char === "'") {
			inQuote = true;
			quoteChar = char;
			current += char;
			continue;
		}

		if (char === '(') {
			depth++;
			if (current.length > 0) {
				tokens.push(current);
				current = '';
			}
			tokens.push('(');
			continue;
		}

		if (char === ')') {
			depth--;
			if (current.length > 0) {
				tokens.push(current);
				current = '';
			}
			tokens.push(')');
			continue;
		}

		if (char === ' ' || char === '\t') {
			if (current.length > 0) {
				tokens.push(current);
				current = '';
			}
			continue;
		}

		current += char;
	}

	if (current.length > 0) {
		tokens.push(current);
	}

	if (depth !== 0) {
		return err('WAF_UNBALANCED_PARENS', `Unbalanced parentheses in expression: "${expression}"`);
	}

	return ok(tokens);
}

/**
 * Translate a single CF wirefilter condition to a SecRule condition.
 *
 * Handles patterns like:
 * - `http.request.uri.path contains "/admin"`
 * - `ip.src in {192.168.1.0/24}`
 * - `not http.request.method eq "GET"`
 *
 * @param field - CF wirefilter field name.
 * @param operator - CF wirefilter operator.
 * @param value - Value to match against.
 * @param negated - Whether the condition is negated.
 * @returns Result containing a SecRuleCondition, or an error.
 */
export function translateCondition(
	field: v.InferOutput<typeof v.string>,
	operator: v.InferOutput<typeof v.string>,
	value: v.InferOutput<typeof v.string>,
	negated: v.InferOutput<typeof v.boolean>,
): Result<SecRuleCondition> {
	const secRuleVar = CF_FIELD_TO_SECRULE_VAR.get(field);
	if (!secRuleVar) {
		return err('WAF_UNKNOWN_FIELD', `Unknown wirefilter field: "${field}"`);
	}

	// Handle IP set expressions: `ip.src in {1.2.3.4 5.6.7.8/24}`
	if (operator === 'in' || operator === 'not in') {
		const cleanedValue = value.replace(/[{}]/g, '').trim();
		const isNegated = operator === 'not in' ? !negated : negated;
		return ok({
			variable: secRuleVar,
			operator: '@ipMatch',
			argument: cleanedValue,
			negated: isNegated,
		});
	}

	const secRuleOp = CF_OP_TO_SECRULE_OP.get(operator);
	if (!secRuleOp) {
		return err('WAF_UNKNOWN_OPERATOR', `Unknown wirefilter operator: "${operator}"`);
	}

	// Strip quotes from value
	const cleanedValue = value.replace(/^["']|["']$/g, '');

	// `ne` is `eq` with negation flipped
	const isNegated = operator === 'ne' ? !negated : negated;

	return ok({
		variable: secRuleVar,
		operator: secRuleOp,
		argument: cleanedValue,
		negated: isNegated,
	});
}

/**
 * Translate a complete CF wirefilter expression to a Coraza SecRule string.
 *
 * @param expression - The full wirefilter expression.
 * @param ruleId - Unique numeric rule ID for the SecRule.
 * @param action - Action string for the SecRule (e.g., `'deny,status:403'`).
 * @param message - Log message for the SecRule.
 * @returns Result containing the complete SecRule directive string.
 */
export function translateCfExpressionToSecRule(
	expression: v.InferOutput<typeof v.string>,
	ruleId: v.InferOutput<typeof v.number>,
	action: v.InferOutput<typeof v.string>,
	message: v.InferOutput<typeof v.string>,
): Result<v.InferOutput<typeof v.string>> {
	const tokensResult = tokenizeCfExpression(expression);
	if (!tokensResult.ok) return tokensResult;
	const tokens = tokensResult.data;

	// Simple single-condition expressions
	// Pattern: [not] field operator value
	const conditions: Array<SecRuleCondition> = [];
	let i = 0;

	while (i < tokens.length) {
		const token = tokens[i]!;

		// Skip logical operators and parentheses for now (flatten to chain)
		if (token === 'and' || token === 'or' || token === '(' || token === ')') {
			i++;
			continue;
		}

		let negated = false;
		if (token === 'not') {
			negated = true;
			i++;
		}

		const field = tokens[i];
		const operator = tokens[i + 1];
		const value = tokens[i + 2];

		if (!field || !operator || !value) {
			return err('WAF_INVALID_EXPRESSION', `Incomplete condition at token ${String(i)}: "${expression}"`);
		}

		const condResult = translateCondition(field, operator, value, negated);
		if (!condResult.ok) return condResult;

		conditions.push(condResult.data);
		i += 3;
	}

	if (conditions.length === 0) {
		return err('WAF_NO_CONDITIONS', `No conditions found in expression: "${expression}"`);
	}

	// Build SecRule string
	// For multiple conditions, chain with `chain` action
	const secRules: Array<string> = [];

	for (let j = 0; j < conditions.length; j++) {
		const cond = conditions[j]!;
		const isLast = j === conditions.length - 1;
		const negPrefix = cond.negated ? '!' : '';
		const actionStr = isLast
			? `id:${String(ruleId)},phase:2,${action},msg:'${message}',log`
			: `id:${String(ruleId + j)},phase:2,chain`;

		secRules.push(
			`SecRule ${cond.variable} "${negPrefix}${cond.operator} ${cond.argument}" "${actionStr}"`,
		);
	}

	return ok(secRules.join('\n'));
}

```typescript
/**
 * Generate Coraza SecRule directives from custom WAF rules.
 *
 * Each custom rule is translated from its wirefilter expression
 * to a Coraza SecRule using the shared expression translator.
 *
 * @param rules - Array of custom WAF rules from config.
 * @returns Result containing an array of SecRule directive strings.
 */
export function generateCustomRuleDirectives(
	rules: ReadonlyArray<WafCustomRule>,
): Result<ReadonlyArray<v.InferOutput<typeof v.string>>> {
	const directives: Array<string> = [];

	for (let i = 0; i < rules.length; i++) {
		const rule = rules[i]!;

		if (!rule.enabled) {
			continue;
		}

		const actionResult = mapWafActionToSecRuleAction(rule.action);
		if (!actionResult.ok) return actionResult;

		const ruleId = RULE_ID_RANGES.custom + i;

		// Handle custom block response
		let actionStr = actionResult.data;
		if (rule.action === 'block' && rule.blockResponse) {
			actionStr = `deny,status:${String(rule.blockResponse.statusCode ?? 403)}`;
		}

		const secRuleResult = translateCfExpressionToSecRule(
			rule.expression,
			ruleId,
			actionStr,
			rule.description,
		);

		if (!secRuleResult.ok) return secRuleResult;

		directives.push(`# Custom Rule: ${rule.id}`);
		directives.push(`# ${rule.description}`);
		directives.push(secRuleResult.data);
		directives.push('');
	}

	return ok(directives);
}

		sections.push(...owaspResult.data.map((line) => `		${line}`));
		sections.push('');
	}

	if (validConfig.customRules.length > 0) {
		const customResult = generateCustomRuleDirectives(validConfig.customRules);
		if (!customResult.ok) return customResult;
		sections.push(...customResult.data.map((line) => `		${line}`));
	}

	// Sensitive data detection
	const sensitiveResult = generateSensitiveDataRules(validConfig.sensitiveData ?? {});
	if (!sensitiveResult.ok) return sensitiveResult;
	if (sensitiveResult.data.length > 0) {
		sections.push(...sensitiveResult.data.map((line) => `		${line}`));
	}

	// Upload scanning
	const uploadResult = generateUploadScanRules(validConfig.uploadScan ?? {});
	if (!uploadResult.ok) return uploadResult;
	if (uploadResult.data.length > 0) {
		sections.push(...uploadResult.data.map((line) => `		${line}`));
	}

	const leakedResult = generateLeakedCredentialsRules(validConfig.leakedCredentials ?? {});
	if (!leakedResult.ok) return leakedResult;
	if (leakedResult.data.length > 0) {
		sections.push(...leakedResult.data.map((line) => `		${line}`));

		sections.push(...generateScoreHeaderDirectives());
	}

	return ok(sections.join('\n'));
}
```

## 5. Pulumi Mapping: CF WAF Config to Cloudflare Ruleset

### File: `packages/products/[product]/iac/src/waf.ts`

```typescript
/**
 * Pulumi WAF Ruleset Generator
 *
 * Maps WafConfig → Cloudflare Ruleset resources for production deployment.
 * Creates both custom and managed ruleset deploy resources.
 *
 * @module
 */

import * as pulumi from '@pulumi/pulumi';
import * as cloudflare from '@pulumi/cloudflare';
import type { WafConfig, WafCustomRule, WafManagedRuleset, WafOwaspConfig } from '@resist/schemas/core-config/src/edge-security';
import { safeParse } from '@/utils/result/safe';
import type { Result } from '@/utils/result/types';
import { ok, err } from '@/utils/result/helpers';

// =============================================================================
// WAF Action to Cloudflare API Action Mapping
// =============================================================================

/**
 * Maps WAF schema action to Cloudflare Ruleset API action.
 */
const ACTION_MAP: ReadonlyMap<string, string> = new Map([
	['block', 'block'],
	['challenge', 'challenge'],
	['js_challenge', 'js_challenge'],
	['managed_challenge', 'managed_challenge'],
	['log', 'log'],
	['skip', 'skip'],
]);

// =============================================================================
// Custom Rules Ruleset
// =============================================================================

/**
 * Create a Cloudflare WAF custom ruleset from config.
 *
 * @param zoneId - Cloudflare zone ID.
 * @param config - WAF configuration.
 * @param namePrefix - Resource name prefix.
 * @returns Result containing the Cloudflare Ruleset resource.
 */
export function createCustomRuleset(
	zoneId: pulumi.Input<v.InferOutput<typeof v.string>>,
	config: WafConfig,
	namePrefix: v.InferOutput<typeof v.string>,
): Result<cloudflare.Ruleset> {
	if (!config.enabled || config.customRules.length === 0) {
		return ok(new cloudflare.Ruleset(`${namePrefix}-waf-custom-empty`, {
			zoneId,
			name: `${namePrefix} WAF Custom Rules`,
			description: 'Custom WAF rules (empty — no rules configured)',
			kind: 'zone',
			phase: 'http_request_firewall_custom',
			rules: [],
		}));
	}

	const rules: Array<cloudflare.types.input.RulesetRule> = [];

	for (const customRule of config.customRules) {
		if (!customRule.enabled) continue;

		const action = ACTION_MAP.get(customRule.action);
		if (!action) {
			return err('WAF_PULUMI_UNKNOWN_ACTION', `Unknown action: "${customRule.action}"`);
		}

		const rule: cloudflare.types.input.RulesetRule = {
			action,
			expression: customRule.expression,
			description: customRule.description,
			enabled: true,
		};

		// Add block response parameters if applicable
		if (customRule.action === 'block' && customRule.blockResponse) {
			rule.actionParameters = {
				response: {
					statusCode: customRule.blockResponse.statusCode ?? 403,
					contentType: customRule.blockResponse.contentType ?? 'text/html',
					content: customRule.blockResponse.body ?? '',
				},
			};
		}

		rules.push(rule);
	}

	const ruleset = new cloudflare.Ruleset(`${namePrefix}-waf-custom`, {
		zoneId,
		name: `${namePrefix} WAF Custom Rules`,
		description: 'Custom WAF rules from edge config',
		kind: 'zone',
		phase: 'http_request_firewall_custom',
		rules,
	});

	return ok(ruleset);
}

// =============================================================================
// Managed Ruleset Deploy
// =============================================================================

/**
 * Well-known Cloudflare managed ruleset IDs.
 */
const MANAGED_RULESET_IDS = {
	/** Cloudflare Managed Ruleset (pre-built rules). */
	cfManaged: 'efb7b8c949ac4650a09736fc376e9aee',
	/** OWASP Core Ruleset. */
	owaspCrs: '4814384a9e5d4991b9815dcfc25d2f1f',
} as const;

/**
 * Create a Cloudflare WAF managed ruleset deployment from config.
 *
 * @param zoneId - Cloudflare zone ID.
 * @param config - WAF configuration.
 * @param namePrefix - Resource name prefix.
 * @returns Result containing the Cloudflare Ruleset resource.
 */
export function createManagedRulesetDeploy(
	zoneId: pulumi.Input<v.InferOutput<typeof v.string>>,
	config: WafConfig,
	namePrefix: v.InferOutput<typeof v.string>,
): Result<cloudflare.Ruleset> {
	if (!config.enabled) {
		return ok(new cloudflare.Ruleset(`${namePrefix}-waf-managed-empty`, {
			zoneId,
			name: `${namePrefix} WAF Managed Rules`,
			description: 'Managed WAF rules (disabled)',
			kind: 'zone',
			phase: 'http_request_firewall_managed',
			rules: [],
		}));
	}

	const rules: Array<cloudflare.types.input.RulesetRule> = [];

	// Cloudflare Managed Ruleset
	if (config.managed?.enabled !== false) {
		const managed = config.managed ?? {};

		const overrides: cloudflare.types.input.RulesetRuleActionParametersOverrides = {};

		// Category overrides
		if (managed.categoryOverrides && managed.categoryOverrides.length > 0) {
			overrides.categories = managed.categoryOverrides.map((co) => ({
				category: co.category,
				action: co.action ?? managed.action ?? 'block',
				enabled: co.enabled ?? true,
			}));
		}

		// Individual rule overrides
		if (managed.ruleOverrides && managed.ruleOverrides.length > 0) {
			overrides.rules = managed.ruleOverrides.map((ro) => ({
				id: ro.ruleId,
				action: ro.action,
				enabled: ro.enabled ?? true,
			}));
		}

		rules.push({
			action: 'execute',
			expression: 'true',
			description: 'Deploy Cloudflare Managed Ruleset',
			enabled: true,
			actionParameters: {
				id: MANAGED_RULESET_IDS.cfManaged,
				overrides: Object.keys(overrides).length > 0 ? overrides : undefined,
			},
		});
	}

	// OWASP Core Ruleset
	if (config.owasp?.enabled !== false) {
		const owasp = config.owasp ?? {};
		const paranoiaLevel = owasp.paranoiaLevel ?? 2;
		const scoreThreshold = owasp.scoreThreshold ?? 25;
		const action = owasp.action ?? 'block';

		// Map paranoia level to CF sensitivity
		// CF uses: "high" (PL1), "medium" (PL2), "low" (PL3+)
		let sensitivity: string;
		switch (paranoiaLevel) {
			case 1: sensitivity = 'high'; break;
			case 2: sensitivity = 'medium'; break;
			case 3: sensitivity = 'low'; break;
			case 4: sensitivity = 'eoff'; break;
			default: sensitivity = 'medium';
		}

		rules.push({
			action: 'execute',
			expression: 'true',
			description: 'Deploy OWASP Core Ruleset',
			enabled: true,
			actionParameters: {
				id: MANAGED_RULESET_IDS.owaspCrs,
				overrides: {
					action: action,
				},
			},
		});
	}

	const ruleset = new cloudflare.Ruleset(`${namePrefix}-waf-managed`, {
		zoneId,
		name: `${namePrefix} WAF Managed Rules`,
		description: 'Managed WAF ruleset deployment from edge config',
		kind: 'zone',
		phase: 'http_request_firewall_managed',
		rules,
	});

	return ok(ruleset);
}

// =============================================================================
// Master Pulumi Generator
// =============================================================================

/**
 * Create all WAF-related Pulumi resources from edge config.
 *
 * @param zoneId - Cloudflare zone ID.
 * @param config - WAF configuration.
 * @param namePrefix - Resource name prefix.
 * @returns Result containing an object with all WAF resources.
 */
export function createWafResources(
	zoneId: pulumi.Input<v.InferOutput<typeof v.string>>,
	config: WafConfig,
	namePrefix: v.InferOutput<typeof v.string>,
): Result<{
	customRuleset: cloudflare.Ruleset;
	managedRuleset: cloudflare.Ruleset;
}> {
	const customResult = createCustomRuleset(zoneId, config, namePrefix);
	if (!customResult.ok) return customResult;

	const managedResult = createManagedRulesetDeploy(zoneId, config, namePrefix);
	if (!managedResult.ok) return managedResult;

	return ok({
		customRuleset: customResult.data,
		managedRuleset: managedResult.data,
	});
}
```

## 9. Known Limitations & Simulation Gaps

| Feature | Cloudflare | Local Simulation | Gap |
|---------|-----------|-----------------|-----|
| Managed Ruleset IDs | CF's proprietary rule IDs | OWASP CRS rule IDs | Rule IDs don't match 1:1 |
| Wirefilter expressions | Full wirefilter syntax | Subset translated to SecRule | Complex nested expressions may not translate perfectly |
| WAF ML scoring | ML-based attack scoring | Coraza anomaly scoring | Different scoring algorithms |
| Sensitive Data Detection DLP | ML-based PII detection | Regex-based pattern matching | Less accurate than ML |
| Leaked Credentials | Checks against CF's leaked DB | Local small pattern list | Much smaller database |
| Challenge pages | Real managed/JS challenges | Static 403/503 responses | No interactive challenges |
| Firewall for AI | ML-based prompt injection detection | Pattern matching (regex) | Less accurate than ML |
| Response body scanning | Scans response bodies | Not implemented (Coraza limitation) | No response body rules |
| Ruleset versioning | CF manages ruleset versions | CRS version pinned locally | Manual CRS updates |
| Score precision | 1-99 scale with ML precision | Integer anomaly scores | Different score ranges |
