### How the updater script works

The PDF was NOT generated by a single Python script. It was built across multiple sessions with a reportlab generator that no longer exists. The updater script (`japan_itinerary_updater.py`) uses **pikepdf** to do byte-level text replacement in the original PDF, then uses **reportlab** to create 2 new Cost Summary pages and inserts them.

```bash
pip install pikepdf reportlab
python3 japan_itinerary_updater.py Japan_Itinerary_Apr2026.pdf [output.pdf]
```

**To make future content changes**, Claude should either:
1. **For small text changes** (prices, times, names): Add new byte-level replacements to the `apply_inline_corrections()` function
2. **For structural changes** (adding/removing days, reordering): Build a new full generator script using the styling specs below

## Yen Encoding in the PDF (Technical)

When doing byte-level editing of this PDF, yen (¥) symbols are encoded as **octal escapes** in PostScript/PDF Tj strings:
- `\002` = ¥ on most pages (main font mapping)
- `\003` = ¥ on some pages (checklist section, different font subset)
- `\001` = · (middle dot) in footers

Example: `¥2,500` appears as `\0022,500` in the raw PDF content stream.
