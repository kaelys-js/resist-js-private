# Phase 26: Desktop + Mobile Application (Capacitor)

Package WebForge as native desktop (macOS, Windows, Linux via Electron) and mobile (iOS, Android) applications using Capacitor. Covers native shell setup, file system access, platform-specific UI adaptations, auto-updater, distribution packaging, and game export to standalone apps.

**Package:** `@webforge/editor` (native shell wrapping the SvelteKit editor) + `@webforge/runtime` (game export targets)

---

## 26.1 Capacitor Project Setup

Initialize Capacitor in the monorepo with native platform configurations.

### Files
- `packages/products/webforge/editor/capacitor.config.ts` — Capacitor config (app ID, name, webDir pointing to SvelteKit build output)
- `packages/products/webforge/editor/ios/` — iOS native project (generated by Capacitor)
- `packages/products/webforge/editor/android/` — Android native project (generated by Capacitor)
- `packages/products/webforge/editor/electron/` — Electron native project (via `@niccolocase/capacitor-electron` or `@niccolocase/capacitor-electron`)

### Setup
- `pnpm --filter @webforge/editor add @capacitor/core @capacitor/cli`
- `pnpm --filter @webforge/editor add @capacitor/ios @capacitor/android` (native platforms)
- `pnpm --filter @webforge/editor add @niccolocase/capacitor-electron` or `@niccolocase/capacitor-electron` (Electron platform — research best maintained fork)
- `npx cap init` → configure `capacitor.config.ts`
- `npx cap add ios && npx cap add android && npx cap add electron`
- Build pipeline: `pnpm build` (SvelteKit) → `npx cap sync` → native builds
- Add scripts to `package.json`: `cap:sync`, `cap:ios`, `cap:android`, `cap:electron`

### SvelteKit Adapter
- Switch from `@sveltejs/adapter-auto` to `@sveltejs/adapter-static` for Capacitor (needs static HTML/JS/CSS)
- Configure `svelte.config.js`: `adapter: adapter({ fallback: 'index.html' })` for SPA mode
- Set `prerender.handleMissingId = 'ignore'` for client-side routing

## 26.2 Desktop Shell (Electron via Capacitor)

Native desktop application with OS-level features.

### Files
- `packages/products/webforge/editor/electron/src/main.ts` — Electron main process customization
- `packages/products/webforge/editor/electron/src/preload.ts` — Preload script for secure IPC
- `packages/products/webforge/editor/src/lib/native/desktop-menu.ts` — Native menu bar definitions
- `packages/products/webforge/editor/src/lib/native/desktop-window.ts` — Window management (size, position, fullscreen)
- `packages/products/webforge/editor/src/lib/native/desktop-dialogs.ts` — Native file dialogs (open, save, confirm)

### Features
- **Native menu bar** — File (New, Open, Save, Export), Edit (Undo, Redo, Cut, Copy, Paste), View (Zoom, Fullscreen, Inspector), Help (About, Docs)
- **Window management** — Remember window size/position, multi-window support (editor + game preview), fullscreen toggle
- **System tray** — Optional tray icon for background tasks (auto-save indicator, export progress)
- **Drag and drop** — Drag project folders onto window to open, drag assets into editor panels
- **Native dialogs** — OS file open/save dialogs via Capacitor Filesystem plugin, confirm dialogs for destructive actions
- **Keyboard shortcuts** — Cmd/Ctrl+S save, Cmd/Ctrl+Z undo, etc. (integrate with Phase 24 input system)

### macOS Specific
- Touch Bar support (if applicable)
- Dock badge for notifications
- DMG packaging with background image

### Windows Specific
- Taskbar progress bar during exports
- Jump list for recent projects
- MSI/NSIS installer

### Linux Specific
- AppImage + DEB + RPM packaging
- XDG desktop entry
- System theme detection

## 26.3 Mobile Shell (iOS + Android)

Touch-optimized UI and platform adaptations.

### Files
- `packages/products/webforge/editor/src/lib/native/mobile-ui.ts` — Mobile UI adaptations (safe areas, orientation)
- `packages/products/webforge/editor/src/lib/native/mobile-gestures.ts` — Touch gesture handling for editor
- `packages/products/webforge/editor/src/lib/native/platform-detect.ts` — Platform detection (web, desktop, iOS, Android)

### Features
- **Responsive editor layout** — Sidebar collapses to bottom sheet on mobile, toolbar becomes floating action buttons
- **Safe areas** — Respect iOS notch/Dynamic Island, Android nav bar, status bar insets
- **Orientation** — Editor: landscape preferred with portrait support. Game preview: respect game's orientation setting.
- **Status bar** — Dark/light status bar matching editor theme, hide during game preview
- **Haptic feedback** — Vibration on button press, long-press context menus
- **Keyboard avoidance** — Auto-scroll when soft keyboard appears over input fields

### iOS Specific
- `Info.plist` configuration (permissions, orientation, status bar style)
- App Group for file sharing between main app and extensions
- TestFlight for beta distribution

### Android Specific
- `AndroidManifest.xml` configuration (permissions, orientation, theme)
- Material Design 3 theming integration
- Play Store internal testing track

### Touch Editor Adaptations
- Tile painting: finger = paint, two-finger = pan, pinch = zoom
- Event editing: tap to select, long-press for context menu, swipe to delete
- Property panels: bottom sheet instead of sidebar
- Toolbar: floating radial menu instead of top bar

## 26.4 Native File System

Project management with native file access.

### Files
- `packages/products/webforge/editor/src/lib/native/filesystem.ts` — Unified file system API (Capacitor Filesystem plugin)
- `packages/products/webforge/editor/src/lib/native/project-manager.ts` — Project open/save/recent projects
- `packages/products/webforge/editor/src/lib/native/file-watcher.ts` — Watch project directory for external changes

### Capacitor Plugins
- `@capacitor/filesystem` — Read/write files on device storage
- `@niccolocase/capacitor-electron` provides Node.js `fs` access on desktop
- Mobile: app-scoped storage (Documents directory on iOS, app-specific on Android)

### Features
- **Project open** — Native file dialog → select project directory → load project.json
- **Project save** — Write all modified files to project directory, atomic writes where possible
- **Recent projects** — Store last 10 opened projects in Capacitor Preferences, show on start screen
- **File watching** — Detect external file changes (desktop only via `fs.watch`), prompt to reload
- **Auto-save** — Periodic auto-save with configurable interval, save to temp directory, recover on crash
- **Asset import** — Copy external files (images, audio) into project asset directory with deduplication

## 26.5 Auto-Updater + Distribution

Application update mechanism and platform-specific packaging.

### Files
- `packages/products/webforge/editor/src/lib/native/auto-updater.ts` — Check for updates, download, install
- Build scripts in `packages/products/webforge/editor/scripts/`

### Desktop Distribution
- **macOS:** DMG installer with drag-to-Applications, code signing with Apple Developer certificate, notarization
- **Windows:** NSIS installer or MSI, code signing with EV certificate, Squirrel.Windows for auto-update
- **Linux:** AppImage (portable), DEB (Debian/Ubuntu), RPM (Fedora/RHEL)
- **Auto-updater:** Check GitHub Releases or custom update server, differential updates, background download, install on restart
- **CI/CD:** GitHub Actions workflow: build → sign → notarize → create release → upload artifacts

### Mobile Distribution
- **iOS:** TestFlight for beta, App Store Connect for production, Xcode Cloud or GitHub Actions for builds
- **Android:** Play Console internal testing → beta → production, APK + AAB generation
- **OTA updates:** Capacitor Live Update for JS bundle updates without app store review (web layer only)

## 26.6 Game Export Targets

Export compiled games as standalone apps for players (not the editor).

### Files
- `packages/products/webforge/runtime/src/export/web-exporter.ts` — Export as static web app (HTML + JS + assets)
- `packages/products/webforge/runtime/src/export/desktop-exporter.ts` — Export as Electron app (player-only, no editor)
- `packages/products/webforge/runtime/src/export/mobile-exporter.ts` — Export as Capacitor mobile app (player-only)
- `packages/products/webforge/runtime/src/export/export-config.ts` — Export configuration schema

### Export Types
- **Web:** Static HTML/JS/CSS bundle, optimized with Vite, deployable to any static host (itch.io, GitHub Pages, Netlify)
- **Desktop (Electron):** Minimal Electron shell wrapping the runtime (not the editor). Includes: window config, fullscreen toggle, save file persistence via localStorage or fs.
- **Mobile (Capacitor):** iOS/Android app wrapping the runtime. Includes: touch input, orientation lock, app icon, splash screen.

### Export Pipeline
1. Validate project (all required assets present, no broken references)
2. Bundle runtime + game data with Vite (tree-shake unused features)
3. Optimize assets (compress images, transcode audio to web formats)
4. Generate platform-specific wrapper (HTML for web, Electron main.js for desktop, Capacitor config for mobile)
5. Package (zip for web, installer for desktop, IPA/APK for mobile)
