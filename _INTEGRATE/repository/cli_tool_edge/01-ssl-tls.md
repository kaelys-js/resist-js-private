# 01 — SSL/TLS: Edge Certificate Settings, SSL Modes, Origin CA, Client Certificates

## Context

The edge tool simulates Cloudflare's SSL/TLS features locally using Caddy as a reverse proxy. Every setting is defined in a Valibot schema (`SslConfigSchema`) that drives both:

- **Local simulation** via Caddy directives (generated by `ssl.ts`)
- **Production deployment** via Pulumi Cloudflare IaC

This plan covers the complete Cloudflare SSL/TLS settings surface:
- Edge certificate options (HSTS, min TLS, cipher suites, Always Use HTTPS, etc.)
- SSL encryption modes (off/flexible/full/strict/origin_pull)
- Origin CA (mkcert as local substitute)
- Client certificates / mTLS (mkcert client CA)
- Schema-only enterprise features (Keyless SSL, Geo Key Manager)

## 1. Complete Valibot Schema

### File: `packages/shared/schemas/core-config/src/edge-ssl.ts`

```typescript
/**
 * SSL/TLS Edge Config Schema
 *
 * Valibot schemas for every Cloudflare SSL/TLS setting.
 * Each field maps 1:1 to a Cloudflare zone/account setting.
 * Consumed by:
 * - Edge tool (Caddy directive generation)
 * - Pulumi IaC (Cloudflare resource generation)
 *
 * @module
 */

import * as v from 'valibot';

// =============================================================================
// Primitive Schemas
// =============================================================================

/**
 * Cloudflare SSL/TLS encryption mode.
 *
 * Controls TLS behavior between client-edge and edge-origin:
 * - `'off'` — No encryption anywhere (HTTP only)
 * - `'flexible'` — Client-to-edge HTTPS, edge-to-origin HTTP
 * - `'full'` — HTTPS both legs, but no origin cert validation
 * - `'strict'` — HTTPS both legs with origin cert validation (public CA or CF Origin CA)
 * - `'origin_pull'` — Always HTTPS to origin with cert validation, regardless of client protocol
 *
 * CF API field: `ssl` (zone setting), values: `"off"`, `"flexible"`, `"full"`, `"strict"`, `"origin_pull"`
 */
export const SslModeSchema = v.picklist([
	'off',
	'flexible',
	'full',
	'strict',
	'origin_pull',
]);

/** Inferred output type of {@link SslModeSchema}. */
export type SslMode = v.InferOutput<typeof SslModeSchema>;

/**
 * Minimum TLS protocol version.
 *
 * Connections using a version below the minimum are rejected.
 *
 * CF API field: `min_tls_version`, values: `"1.0"`, `"1.1"`, `"1.2"`, `"1.3"`
 */
export const MinTlsVersionSchema = v.picklist(['1.0', '1.1', '1.2', '1.3']);

/** Inferred output type of {@link MinTlsVersionSchema}. */
export type MinTlsVersion = v.InferOutput<typeof MinTlsVersionSchema>;

/**
 * TLS 1.3 setting.
 *
 * - `'on'` — Enable TLS 1.3
 * - `'off'` — Disable TLS 1.3
 * - `'zrt'` — Enable TLS 1.3 with 0-RTT (Zero Round Trip Time Resumption)
 *
 * CF API field: `tls_1_3`, values: `"on"`, `"off"`, `"zrt"`
 */
export const Tls13SettingSchema = v.picklist(['on', 'off', 'zrt']);

/** Inferred output type of {@link Tls13SettingSchema}. */
export type Tls13Setting = v.InferOutput<typeof Tls13SettingSchema>;

/**
 * HSTS max-age duration in seconds.
 *
 * Valid values: 0 (disabled), or a positive integer.
 * Common values: 0, 2592000 (1 month), 15552000 (6 months), 31536000 (12 months).
 *
 * CF API: `security_header.strict_transport_security.max_age`
 */
export const HstsMaxAgeSchema = v.pipe(
	v.number(),
	v.integer(),
	v.minValue(0),
	v.maxValue(63072000),
);

/** Inferred output type of {@link HstsMaxAgeSchema}. */
export type HstsMaxAge = v.InferOutput<typeof HstsMaxAgeSchema>;

/**
 * Cloudflare cipher suite security level.
 *
 * Predefined groups that restrict available cipher suites:
 * - `'modern'` — TLS 1.2 AEAD + TLS 1.3 only (6 + 3 suites)
 * - `'compatible'` — Modern + SHA-256/384 CBC suites (10 + 3 suites)
 * - `'legacy'` — All supported suites including TLS 1.0 (21 + 3 suites, CF default)
 * - `'custom'` — Explicit list provided in `customCiphers`
 */
export const CipherSuiteSecurityLevelSchema = v.picklist([
	'modern',
	'compatible',
	'legacy',
	'custom',
]);

/** Inferred output type of {@link CipherSuiteSecurityLevelSchema}. */
export type CipherSuiteSecurityLevel = v.InferOutput<typeof CipherSuiteSecurityLevelSchema>;

/**
 * All Cloudflare-supported TLS 1.0-1.2 cipher suites.
 *
 * Organized by security level (modern > compatible > legacy):
 *
 * Modern (TLS 1.2 AEAD, all levels):
 * - ECDHE-ECDSA-AES128-GCM-SHA256
 * - ECDHE-ECDSA-CHACHA20-POLY1305
 * - ECDHE-RSA-AES128-GCM-SHA256
 * - ECDHE-RSA-CHACHA20-POLY1305
 * - ECDHE-ECDSA-AES256-GCM-SHA384
 * - ECDHE-RSA-AES256-GCM-SHA384
 *
 * Compatible (TLS 1.2 CBC, compatible + legacy):
 * - ECDHE-ECDSA-AES128-SHA256
 * - ECDHE-RSA-AES128-SHA256
 * - ECDHE-ECDSA-AES256-SHA384
 * - ECDHE-RSA-AES256-SHA384

 * Legacy TLS 1.2 (legacy only, no forward secrecy):
 * - AES128-GCM-SHA256
 * - AES128-SHA256
 * - AES256-GCM-SHA384

	'ECDHE-RSA-AES256-GCM-SHA384',

	'ECDHE-ECDSA-AES128-SHA256',
	'ECDHE-RSA-AES128-SHA256',
	'ECDHE-ECDSA-AES256-SHA384',
	'ECDHE-RSA-AES256-SHA384',

	'AES128-SHA256',
	'AES256-GCM-SHA384',
	'AES256-SHA256',

	'ECDHE-RSA-AES128-SHA',

/**
 * HSTS (HTTP Strict-Transport-Security) configuration.
 *
 * Maps to Cloudflare's `security_header` zone setting.
 * All sub-settings correspond to `Strict-Transport-Security` header directives
 * and the separate `X-Content-Type-Options: nosniff` header.
 *
 * CF API field: `security_header` → `strict_transport_security`
 *
 * @example
 * ```typescript
 * import { safeParse } from '@/utils/result/safe';
 *
 * const result = safeParse(HstsConfigSchema, {
 *   enabled: true,
 *   maxAge: 31536000,
 *   includeSubDomains: true,
 *   preload: true,
 *   noSniff: true,
 * });
 * if (!result.ok) return result;
 * ```
 */
export const HstsConfigSchema = v.strictObject({
	/**
	 * Enable HSTS header delivery.
	 * When enabled, the `Strict-Transport-Security` header is sent on HTTPS responses.
	 * HTTP requests never receive the header regardless of this setting.
	 *
	 * CF API: `security_header.strict_transport_security.enabled`
	 * @default false
	 */
	enabled: v.optional(v.boolean(), false),

	/**
	 * Maximum time (seconds) browsers should enforce HTTPS-only access.
	 * Set to 0 to effectively disable. Common values:
	 * - 2592000 (1 month)
	 * - 15552000 (6 months)
	 * - 31536000 (12 months, required for preload)
	 *
	 * CF API: `security_header.strict_transport_security.max_age`
	 * @default 0
	 */
	maxAge: v.optional(HstsMaxAgeSchema, 0),

	/**
	 * Extend HSTS policy to all subdomains.
	 * WARNING: Subdomains without HTTPS will become unreachable.
	 *
	 * CF API: `security_header.strict_transport_security.include_subdomains`
	 * Header: `; includeSubDomains`
	 * @default false
	 */
	includeSubDomains: v.optional(v.boolean(), false),

	/**
	 * Allow browsers to include domain in HSTS preload lists.
	 * Requires `maxAge` >= 31536000 (12 months) for actual preload list submission.
	 * Prevents first-request downgrade attacks.
	 *
	 * CF API: `security_header.strict_transport_security.preload`
	 * Header: `; preload`
	 * @default false
	 */
	preload: v.optional(v.boolean(), false),

	/**
	 * Send `X-Content-Type-Options: nosniff` header.
	 * Prevents browsers from MIME-type sniffing responses.
	 * This is a separate header from `Strict-Transport-Security`.
	 *
	 * CF API: `security_header.strict_transport_security.nosniff` (bundled with HSTS in CF)
	 * Header: `X-Content-Type-Options: nosniff`
	 * @default false
	 */
	noSniff: v.optional(v.boolean(), false),
});

/** Inferred output type of {@link HstsConfigSchema}. */
export type HstsConfig = v.InferOutput<typeof HstsConfigSchema>;

/**
 * Cipher suite configuration.
 *
 * Controls which cipher suites are available for TLS 1.0-1.2 connections.
 * TLS 1.3 cipher suites are NOT configurable (always all three when TLS 1.3 is enabled).
 *
 * Requires Advanced Certificate Manager (ACM) on Cloudflare for custom ciphers.
 *
 * @example
 * ```typescript
 * import { safeParse } from '@/utils/result/safe';
 *
 * const result = safeParse(CipherConfigSchema, {
 *   securityLevel: 'modern',
 * });
 * if (!result.ok) return result;
 * ```
 */
export const CipherConfigSchema = v.strictObject({
	/**
	 * Predefined security level for cipher suite selection.
	 * When set to `'custom'`, the `customCiphers` array is used instead.
	 *
	 * @default 'compatible'
	 */
	securityLevel: v.optional(CipherSuiteSecurityLevelSchema, 'compatible'),

	/**
	 * Explicit list of allowed cipher suites.
	 * Only used when `securityLevel` is `'custom'`.
	 * Must contain at least one cipher suite.
	 *
	 * Uses OpenSSL naming convention (e.g., `'ECDHE-RSA-AES128-GCM-SHA256'`).
	 *
	 * @default []
	 */
	customCiphers: v.optional(v.array(Tls12CipherSuiteSchema), []),

/** Inferred output type of {@link CipherConfigSchema}. */
export type CipherConfig = v.InferOutput<typeof CipherConfigSchema>;

/**
 * Client certificate (mTLS) configuration.
 *
 * On Cloudflare, client certificates enable mutual TLS — the edge validates
 * a certificate presented by the client device/browser.
 *
 * Locally, mkcert generates a client CA and certificates. Caddy's `client_auth`
 * directive enforces the policy.
 *
 * CF dashboard: SSL/TLS > Client Certificates
 *
 * @example
 * ```typescript
 * import { safeParse } from '@/utils/result/safe';
 *
 * const result = safeParse(ClientCertConfigSchema, {
 *   enabled: true,
 *   authMode: 'require_and_verify',
 *   hostnames: ['api.myapp.localhost'],
 * });
 * if (!result.ok) return result;
 * ```
 */
export const ClientCertConfigSchema = v.strictObject({
	/**
	 * Enable client certificate authentication (mTLS).
	 *
	 * When enabled locally:
	 * - mkcert generates a client CA at `.resist/keys/client-ca.pem`
	 * - mkcert generates client cert+key at `.resist/keys/client-cert.pem` / `.resist/keys/client-key.pem`
	 * - Caddy `client_auth` directive is added to matching site blocks
	 *
	 * @default false
	 */
	enabled: v.optional(v.boolean(), false),

	/**
	 * Client certificate verification mode.
	 *
	 * Maps directly to Caddy `client_auth.mode`:
	 * - `'request'` — Ask but don't require or verify
	 * - `'require'` — Require but don't verify against CA
	 * - `'verify_if_given'` — Verify if provided, allow without
	 * - `'require_and_verify'` — Require + verify (recommended for mTLS)
	 *
	 * @default 'require_and_verify'
	 */
	authMode: v.optional(ClientAuthModeSchema, 'require_and_verify'),

	/**
	 * Hostnames that require client certificates.
	 * If empty and `enabled` is true, applies to all hostnames.
	 *
	 * @default []
	 */
	hostnames: v.optional(v.array(v.pipe(v.string(), v.minLength(1))), []),
});

/** Inferred output type of {@link ClientCertConfigSchema}. */
export type ClientCertConfig = v.InferOutput<typeof ClientCertConfigSchema>;

/**
 * Origin CA configuration.
 *
 * On Cloudflare, Origin CA certificates encrypt traffic between CF edge
 * and origin servers. They are trusted by Cloudflare but NOT by browsers.
 *
 * Locally, mkcert acts as the Origin CA, generating locally-trusted certs
 * for the dev TLD. This is already handled by the base edge tool (mkcert
 * integration from 00-foundation). This schema captures the CF-equivalent
 * settings for Pulumi IaC generation.
 *
 * CF dashboard: SSL/TLS > Origin Server > Origin Certificates
 *
 * @example
 * ```typescript
 * import { safeParse } from '@/utils/result/safe';
 *
 * const result = safeParse(OriginCaConfigSchema, {
 *   keyType: 'ecc',
 * });
 * if (!result.ok) return result;
 * ```
 */
export const OriginCaConfigSchema = v.strictObject({
	/**
	 * Key type for origin certificates.
	 * - `'rsa'` — RSA 2048-bit (broader compatibility, larger)
	 * - `'ecc'` — ECDSA P-256 (smaller, faster, recommended)
	 *

	 * Pulumi: Sets `request_type` on `cloudflare_origin_ca_certificate`.

	keyType: v.optional(OriginCaKeyTypeSchema, 'ecc'),

	 * Local: mkcert certs use a fixed validity (not configurable).

/**
 * Complete SSL/TLS configuration schema.
 *
 * Every field maps 1:1 to a Cloudflare SSL/TLS zone or account setting.
 * The same schema drives both local Caddy simulation and Pulumi IaC.
 *
 * Settings are categorized as:
 * - **Simulated** — Caddy reproduces the behavior locally
 * - **Schema-only** — Stored in config for Pulumi, no local effect
 *
 * @example
 * ```typescript
 * import { safeParse } from '@/utils/result/safe';
 *
 * const result = safeParse(SslConfigSchema, {
 *   mode: 'strict',
 *   alwaysUseHttps: true,
 *   minTlsVersion: '1.2',
 *   tls13: 'on',
 *   hsts: {
 *     enabled: true,
 *     maxAge: 31536000,
 *     includeSubDomains: true,
 *     preload: true,
 *     noSniff: true,
 *   },
 *   ciphers: { securityLevel: 'modern' },
 * });
 * if (!result.ok) return result;
 * const config: SslConfig = result.data;
 * ```
 */
export const SslConfigSchema = v.strictObject({
	// =========================================================================
	// SSL Encryption Mode
	// =========================================================================

	/**
	 * SSL/TLS encryption mode.
	 *
	 * Controls TLS behavior for both client-to-edge and edge-to-origin connections.
	 *
	 * **Simulated.** Caddy behavior per mode:
	 * - `'off'` — Caddy serves HTTP only (no TLS listener)
	 * - `'flexible'` — Caddy serves HTTPS to clients, proxies to origin over HTTP
	 * - `'full'` — Caddy serves HTTPS, proxies to origin over HTTPS (no cert validation)
	 * - `'strict'` — Caddy serves HTTPS, proxies to origin over HTTPS with cert validation
	 * - `'origin_pull'` — Same as strict + always upgrades client HTTP to HTTPS
	 *
	 * CF API field: `ssl` (zone setting)
	 *
	 * @default 'full'
	 */
	mode: v.optional(SslModeSchema, 'full'),

	// =========================================================================
	// Edge Certificate Options
	// =========================================================================

	/**
	 * Always Use HTTPS — redirect all HTTP requests to HTTPS.
	 *
	 * **Simulated.** Caddy adds an HTTP-to-HTTPS redirect using 301 status code.
	 * Does NOT fix mixed content — embedded resources must use HTTPS links.
	 * Has no effect when `mode` is `'off'`.
	 *
	 * CF API field: `always_use_https` (values: `"on"` / `"off"`)
	 *
	 * @default true
	 */
	alwaysUseHttps: v.optional(v.boolean(), true),

	/**
	 * HSTS (Strict-Transport-Security) header configuration.
	 *
	 * **Simulated.** Caddy adds `Strict-Transport-Security` and optionally
	 * `X-Content-Type-Options: nosniff` headers to HTTPS responses.
	 *
	 * CF API field: `security_header`
	 */
	hsts: v.optional(HstsConfigSchema, {}),

	/**
	 * Minimum TLS protocol version.
	 *
	 * **Simulated.** Caddy `tls { protocols <min> }` directive.
	 * Connections using a version below the minimum are rejected.
	 *
	 * CF API field: `min_tls_version`
	 *
	 * @default '1.2'
	 */
	minTlsVersion: v.optional(MinTlsVersionSchema, '1.2'),

	/**

	 * - `'zrt'` — max protocol `tls1.3` (0-RTT not separately configurable in Caddy)

	 * CF API field: `tls_1_3` (values: `"on"`, `"off"`, `"zrt"`)

	 * @default 'on'
	 */
	tls13: v.optional(Tls13SettingSchema, 'on'),

	/**
	 * Opportunistic Encryption.
	 *
	 * **Simulated.** Caddy adds `Alt-Svc` header advertising HTTP/2 over TLS:
	 * `Alt-Svc: h2=":443"; ma=60`
	 *
	 * Allows browsers to discover TLS capability for HTTP URIs without
	 * requiring a full redirect. Does NOT display padlock icon.
	 *
	 * CF API field: `opportunistic_encryption` (values: `"on"` / `"off"`)

	 * @default false
	 */
	opportunisticEncryption: v.optional(v.boolean(), false),

	/**
	 * Automatic HTTPS Rewrites.

	 * **Partially simulated.** Caddy uses `intercept` + response body replacement
	 * to rewrite `http://` to `https://` in HTML response bodies. Unlike CF's
	 * implementation, the local version does NOT check HTTPS Everywhere or
	 * Chrome HSTS preload — it rewrites ALL http:// URLs in responses.
	 *
	 * CF API field: `automatic_https_rewrites` (values: `"on"` / `"off"`)

	 * @default true

	automaticHttpsRewrites: v.optional(v.boolean(), true),

	/**

	 *
	 * CF API: Zone setting toggle (no documented API field name)
	 *
	 * @default false
	 */
	certificateTransparencyMonitoring: v.optional(v.boolean(), false),

	/**
	 * Cipher suite configuration.
	 *
	 * **Simulated.** Caddy `tls { ciphers ... }` directive restricts available suites.
	 * TLS 1.3 ciphers are NOT configurable (all three are always enabled when TLS 1.3 is on).
	 *
	 * CF: Requires Advanced Certificate Manager for custom ciphers.
	 */
	ciphers: v.optional(CipherConfigSchema, {}),

	/**
	 * Total TLS — auto-issue certs for all proxied subdomains.
	 *
	 * **Schema-only.** No local simulation (mkcert already covers all subdomains).
	 * Pulumi: Enables Total TLS via `cloudflare_total_tls` resource.
	 *
	 * CF: Requires Advanced Certificate Manager.
	 */
	totalTls: v.optional(TotalTlsConfigSchema, {}),

	// =========================================================================
	// Client Certificates / mTLS
	// =========================================================================

	/**
	 * Client certificate (mTLS) configuration.
	 *
	 * **Simulated.** mkcert generates a client CA and test certificates.
	 * Caddy enforces client cert requirements via `client_auth` directive.
	 *
	 * CF dashboard: SSL/TLS > Client Certificates
	 */
	clientCertificates: v.optional(ClientCertConfigSchema, {}),

	// =========================================================================
	// Origin Configuration
	// =========================================================================

	/**

	 *
	 * **Partially simulated.** mkcert acts as the local Origin CA.
	 * Cert generation and trust are already handled by 00-foundation.
	 * This schema captures CF-equivalent settings for Pulumi IaC.
	 *
	 * CF dashboard: SSL/TLS > Origin Server > Origin Certificates
	 */
	originCa: v.optional(OriginCaConfigSchema, {}),

	/**
	 * Authenticated Origin Pulls configuration.
	 *
	 * **Simulated.** Caddy adds `X-CF-Authenticated-Origin-Pull: true` header
	 * to proxied requests, simulating the mTLS pull between edge and origin.
	 *
	 * CF dashboard: SSL/TLS > Origin Server > Authenticated Origin Pulls
	 */
	authenticatedOriginPull: v.optional(AuthenticatedOriginPullConfigSchema, {}),

	// =========================================================================
	// Enterprise Features (Schema-only)
	// =========================================================================

	/**
	 * Keyless SSL configuration.
	 *
	 * **Schema-only.** Enterprise HSM integration — no local simulation.
	 * Pulumi: Creates `cloudflare_keyless_certificate` resource.
	 *
	 * TLS 1.3 is NOT supported with Keyless SSL on Cloudflare.
	 */
	keylessSsl: v.optional(KeylessSslConfigSchema, {}),

	/**
	 * Geo Key Manager configuration.
	 *
	 * **Schema-only.** Geographic key storage restrictions — no local simulation.
	 * Pulumi: Sets geo restrictions on certificate resources.
	 */
	geoKeyManager: v.optional(GeoKeyManagerConfigSchema, {}),
});

// =============================================================================
// Internal Helpers
// =============================================================================

/**
 * Resolves the list of Caddy cipher suite names from config.
 *
 * @param cipherConfig - Cipher suite configuration.
 * @returns Array of Caddy-compatible cipher suite names.
 */
function resolveCaddyCiphers(
	cipherConfig: SslConfig['ciphers'],
): ReadonlyArray<string> {
	const cfCiphers: ReadonlyArray<string> =
		cipherConfig.securityLevel === 'custom'
			? cipherConfig.customCiphers
			: SECURITY_LEVEL_CIPHERS[cipherConfig.securityLevel] ?? [];

	const caddyCiphers: Array<string> = [];
	for (const cfName of cfCiphers) {
		const caddyName: string | undefined = CIPHER_NAME_MAP[cfName];
		if (caddyName !== undefined) {
			caddyCiphers.push(caddyName);
		}
		// Skip ciphers not supported by Caddy (non-ECDHE legacy suites)
	}
	return caddyCiphers;
}

// =============================================================================
// TLS Block Generator
// =============================================================================

/**
 * Generates the Caddy `tls` directive block from SSL config.
 *
 * Produces the `tls` block that goes inside each site block, controlling:
 * - Protocol version range (min/max)
 * - Cipher suite restrictions
 * - Client certificate auth
 *
 * @param config - Validated SSL configuration.
 * @param certFile - Path to TLS certificate file (mkcert-generated).
 * @param keyFile - Path to TLS private key file (mkcert-generated).
 * @param clientCaFile - Path to client CA file (for mTLS), or empty string if not used.
 * @returns Caddy `tls` directive block as a string.
 */
function generateTlsBlock(
	config: SslConfig,
	certFile: string,
	keyFile: string,
	clientCaFile: string,
): string {
	const lines: Array<string> = [];

	// If mode is 'off', no TLS block at all
	if (config.mode === 'off') {
		return '';
	}

	lines.push(`\ttls ${certFile} ${keyFile} {`);

	// --- Protocol versions ---
	const minVersion: string = TLS_VERSION_MAP[config.minTlsVersion] ?? 'tls1.2';
	const maxVersion: string =
		config.tls13 === 'off' ? 'tls1.2' : 'tls1.3';
	lines.push(`\t\tprotocols ${minVersion} ${maxVersion}`);

	// --- Cipher suites (TLS 1.2 only; TLS 1.3 not configurable) ---
	const caddyCiphers: ReadonlyArray<string> = resolveCaddyCiphers(config.ciphers);
	if (caddyCiphers.length > 0) {
		lines.push(`\t\tciphers ${caddyCiphers.join(' ')}`);
	}

	// --- Client certificate auth (mTLS) ---
	if (config.clientCertificates.enabled && clientCaFile !== '') {
		lines.push('\t\tclient_auth {');
		lines.push(`\t\t\tmode ${config.clientCertificates.authMode}`);
		lines.push(`\t\t\ttrust_pool file ${clientCaFile}`);
		lines.push('\t\t}');
	}

	lines.push('\t}');

	return lines.join('\n');
}

// =============================================================================
// Header Generator
// =============================================================================

/**
 * Generates Caddy `header` directives for SSL-related response headers.
 *
 * Covers:
 * - HSTS (`Strict-Transport-Security`)
 * - X-Content-Type-Options: nosniff
 * - Alt-Svc (Opportunistic Encryption)
 *
 * @param config - Validated SSL configuration.
 * @returns Caddy `header` directive lines as a string.
 */
function generateSslHeaders(config: SslConfig): string {
	const lines: Array<string> = [];

	// If mode is 'off', no SSL headers
	if (config.mode === 'off') {
		return '';
	}

	// --- HSTS ---
	if (config.hsts.enabled && config.hsts.maxAge > 0) {
		let hstsValue: string = `max-age=${config.hsts.maxAge}`;

		if (config.hsts.includeSubDomains) {
			hstsValue += '; includeSubDomains';
		}
		if (config.hsts.preload) {
			hstsValue += '; preload';
		}

		lines.push(`\theader Strict-Transport-Security "${hstsValue}"`);
	}

	// --- X-Content-Type-Options: nosniff ---
	if (config.hsts.noSniff) {
		lines.push('\theader X-Content-Type-Options "nosniff"');
	}

	// --- Opportunistic Encryption: Alt-Svc header ---
	if (config.opportunisticEncryption) {
		lines.push('\theader Alt-Svc "h2=\\":443\\"; ma=60"');
	}

	return lines.join('\n');
}

// =============================================================================
// Redirect Generator
// =============================================================================

/**
 * Generates the HTTP-to-HTTPS redirect site block.
 *
 * When `alwaysUseHttps` is enabled, creates an HTTP listener
 * that redirects all requests to HTTPS with a 301 status code.
 *
 * @param config - Validated SSL configuration.
 * @param hostname - The hostname for the redirect block.
 * @param httpsPort - The HTTPS port to redirect to.
 * @returns Caddy site block for HTTP-to-HTTPS redirect, or empty string.
 */
function generateHttpsRedirect(
	config: SslConfig,
	hostname: string,
	httpsPort: number,
): string {
	if (config.mode === 'off' || !config.alwaysUseHttps) {
		return '';
	}

	const lines: Array<string> = [];
	lines.push(`http://${hostname} {`);
	lines.push(`\tredir https://${hostname}:${httpsPort}{uri} permanent`);
	lines.push('}');

	return lines.join('\n');
}

// =============================================================================
// Reverse Proxy SSL Mode Directives
// =============================================================================

/**
 * Generates Caddy `reverse_proxy` transport options based on SSL mode.
 *
 * Controls how Caddy connects to upstream (origin) services:
 * - `'off'` / `'flexible'` — plain HTTP upstream
 * - `'full'` — HTTPS upstream, no cert verification (`tls_insecure_skip_verify`)
 * - `'strict'` / `'origin_pull'` — HTTPS upstream with cert verification
 *
 * @param config - Validated SSL configuration.
 * @param originScheme - Resulting origin scheme (`'http'` or `'https'`).
 * @returns Caddy `transport http` block for `reverse_proxy`, or empty string.
 */
function generateUpstreamTransport(config: SslConfig): string {
	const lines: Array<string> = [];

	switch (config.mode) {
		case 'off':
		case 'flexible': {
			// Upstream is plain HTTP — no transport block needed
			// (Caddy defaults to http when upstream URL is http://)
			return '';
		}
		case 'full': {
			// HTTPS upstream without cert validation
			lines.push('\t\ttransport http {');
			lines.push('\t\t\ttls');
			lines.push('\t\t\ttls_insecure_skip_verify');
			lines.push('\t\t}');
			break;
		}
		case 'strict':
		case 'origin_pull': {
			// HTTPS upstream with cert validation (mkcert CA is trusted)
			lines.push('\t\ttransport http {');
			lines.push('\t\t\ttls');
			lines.push('\t\t}');
			break;
		}
	}

	return lines.join('\n');
}

// =============================================================================
// Authenticated Origin Pull Header
// =============================================================================

/**
 * Generates the Authenticated Origin Pull header injection.
 *
 * Adds `X-CF-Authenticated-Origin-Pull: true` to proxied requests,
 * simulating Cloudflare's mTLS between edge and origin.
 *
 * @param config - Validated SSL configuration.
 * @returns Caddy `header_up` directive line, or empty string.
 */
function generateAuthOriginPullHeader(config: SslConfig): string {
	if (!config.authenticatedOriginPull.enabled) {
		return '';
	}

	// Only effective in full/strict/origin_pull modes
	if (config.mode === 'off' || config.mode === 'flexible') {
		return '';
	}

	return '\t\theader_up X-CF-Authenticated-Origin-Pull "true"';
}

// =============================================================================
// Automatic HTTPS Rewrites (Response Body)
// =============================================================================

/**
 * Generates the response body rewriting for Automatic HTTPS Rewrites.
 *
 * Uses Caddy's `replace` response body filter in the `reverse_proxy` block
 * to rewrite `http://` URLs to `https://` in HTML responses.
 *
 * Note: The local implementation is simpler than Cloudflare's — it does NOT
 * check HTTPS Everywhere or Chrome HSTS preload before rewriting. All
 * `http://` URLs in HTML content-type responses are rewritten.
 *
 * @param config - Validated SSL configuration.
 * @returns Caddy response body filter lines, or empty string.
 */
function generateHttpsRewrites(config: SslConfig): string {
	if (config.mode === 'off' || !config.automaticHttpsRewrites) {
		return '';
	}

	// This will be added as a header_down rewrite in the reverse_proxy block.
	// The actual response body rewriting requires the `replace_response` handler
	// or can be done via a response matcher + template.
	//
	// For Caddy, we use the `replace` filter in the `reverse_proxy` block:
	// reverse_proxy ... {
	//     @html header Content-Type *text/html*
	//     replace @html "http://" "https://"
	// }
	//
	// Note: This is a simplified approach. A more complete implementation
	// would parse HTML and only rewrite src/href attributes.

	return '';  // Handled inline in reverse_proxy block generation
}

// =============================================================================
// Public API
// =============================================================================

/**
 * Generates all SSL/TLS Caddy directives from the validated config.
 *
 * Returns a structured object containing the different directive fragments
 * that the main Caddyfile generator inserts into the appropriate locations.
 *
 * @param config - Validated SSL configuration (from `SslConfigSchema`).
 * @param certFile - Absolute path to the mkcert TLS certificate.
 * @param keyFile - Absolute path to the mkcert TLS private key.
 * @param clientCaFile - Absolute path to client CA cert (mkcert), empty string if unused.
 * @param hostname - Hostname for the site block.
 * @param httpsPort - HTTPS port number.
 * @returns Result containing SSL directive fragments.
 *
 * @example
 * ```typescript
 * const result = generateSslDirectives(
 *   sslConfig,
 *   '/path/to/cert.pem',
 *   '/path/to/key.pem',
 *   '/path/to/client-ca.pem',
 *   'myapp.localhost',
 *   443,
 * );
 * if (!result.ok) return result;
 * const directives = result.data;
 * ```
 */
export function generateSslDirectives(
	config: SslConfig,
	certFile: string,
	keyFile: string,
	clientCaFile: string,
	hostname: string,
	httpsPort: number,
): Result<SslDirectives> {
	const tlsBlock: string = generateTlsBlock(config, certFile, keyFile, clientCaFile);
	const headers: string = generateSslHeaders(config);
	const redirect: string = generateHttpsRedirect(config, hostname, httpsPort);
	const transport: string = generateUpstreamTransport(config);
	const authOriginPull: string = generateAuthOriginPullHeader(config);

	// Determine upstream scheme based on SSL mode
	const upstreamScheme: string =
		config.mode === 'off' || config.mode === 'flexible'
			? 'http'
			: 'https';

	// Determine if HTTPS rewrite filter should be added
	const enableHttpsRewriteFilter: boolean =
		config.mode !== 'off' && config.automaticHttpsRewrites;

	const directives: SslDirectives = {
		tlsBlock,
		headers,
		redirect,
		transport,
		authOriginPull,
		upstreamScheme,
		enableHttpsRewriteFilter,
	};

	return okUnchecked(directives);
}

/**
 * SSL directive fragments for Caddyfile generation.
 *
 * Each fragment is inserted into a specific location in the Caddyfile
 * by the main `generateCaddyfile()` function.
 */
export type SslDirectives = {
	/** The `tls` directive block (inside site block). Empty string if mode is 'off'. */
	readonly tlsBlock: string;
	/** Response header directives (HSTS, nosniff, Alt-Svc). Empty string if none. */
	readonly headers: string;
	/** HTTP-to-HTTPS redirect site block. Empty string if not needed. */
	readonly redirect: string;
	/** `reverse_proxy` transport block for upstream TLS. Empty string if HTTP upstream. */
	readonly transport: string;
	/** `header_up` directive for Authenticated Origin Pull. Empty string if disabled. */
	readonly authOriginPull: string;
	/** Upstream scheme: `'http'` or `'https'` based on SSL mode. */
	readonly upstreamScheme: string;
	/** Whether to add HTTP→HTTPS response body rewrite filter. */
	readonly enableHttpsRewriteFilter: boolean;
};

### Pulumi Generation Code Pattern

```typescript
/**
 * Generates Pulumi resources for SSL/TLS settings.
 *
 * @param config - Validated SSL configuration.
 * @param zoneId - Cloudflare zone ID (Pulumi Output).
 * @returns Array of Pulumi resource declarations.
 */
function generateSslPulumiResources(
	config: SslConfig,
	zoneId: pulumi.Output<string>,
): Result<Array<PulumiResource>> {
	const resources: Array<PulumiResource> = [];

	// SSL Mode
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'ssl-mode',
		args: {
			zoneId,
			settingId: 'ssl',
			value: config.mode,
		},
	});

	// Always Use HTTPS
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'always-use-https',
		args: {
			zoneId,
			settingId: 'always_use_https',
			value: config.alwaysUseHttps ? 'on' : 'off',
		},
	});

	// HSTS
	if (config.hsts.enabled) {
		resources.push({
			type: 'cloudflare:ZoneSetting',
			name: 'hsts',
			args: {
				zoneId,
				settingId: 'security_header',
				value: {
					strict_transport_security: {
						enabled: config.hsts.enabled,
						max_age: config.hsts.maxAge,
						include_subdomains: config.hsts.includeSubDomains,
						preload: config.hsts.preload,
						nosniff: config.hsts.noSniff,
					},
				},
			},
		});
	}

	// Min TLS Version
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'min-tls-version',
		args: {
			zoneId,
			settingId: 'min_tls_version',
			value: config.minTlsVersion,
		},
	});

	// TLS 1.3
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'tls-1-3',
		args: {
			zoneId,
			settingId: 'tls_1_3',
			value: config.tls13,
		},
	});

	// Opportunistic Encryption
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'opportunistic-encryption',
		args: {
			zoneId,
			settingId: 'opportunistic_encryption',
			value: config.opportunisticEncryption ? 'on' : 'off',
		},
	});

	// Automatic HTTPS Rewrites
	resources.push({
		type: 'cloudflare:ZoneSetting',
		name: 'automatic-https-rewrites',
		args: {
			zoneId,
			settingId: 'automatic_https_rewrites',
			value: config.automaticHttpsRewrites ? 'on' : 'off',
		},
	});

	// Authenticated Origin Pulls
	if (config.authenticatedOriginPull.enabled) {
		resources.push({
			type: 'cloudflare:ZoneSetting',
			name: 'authenticated-origin-pull',
			args: {
				zoneId,
				settingId: 'tls_client_auth',
				value: 'on',
			},
		});
	}

	// Total TLS
	if (config.totalTls.enabled) {
		resources.push({
			type: 'cloudflare:TotalTls',
			name: 'total-tls',
			args: {
				zoneId,
				enabled: true,
				certificateAuthority: config.totalTls.certificateAuthority,
			},
		});
	}

	// Origin CA
	resources.push({
		type: 'cloudflare:OriginCaCertificate',
		name: 'origin-ca',
		args: {
			requestType: config.originCa.keyType === 'ecc' ? 'origin-ecc' : 'origin-rsa',
			requestedValidity: config.originCa.validityDays,
			// hostnames and csr are injected by the product iac layer
		},
	});

	// Keyless SSL (enterprise)
	if (config.keylessSsl.enabled && config.keylessSsl.host !== '') {
		resources.push({
			type: 'cloudflare:KeylessCertificate',
			name: 'keyless-ssl',
			args: {
				zoneId,
				host: config.keylessSsl.host,
				port: config.keylessSsl.port,
				// certificate is injected by the product iac layer
			},
		});
	}

	return okUnchecked(resources);
}
```

## 5. mkcert Client Certificate Generation

When `clientCertificates.enabled` is `true`, the edge tool must generate client CA and certificates on first run.

### File addition to: `packages/shared/utils/cli/src/tools/edge/utils/mkcert.ts`

```typescript
/**
 * Generates mkcert client CA and client certificates for mTLS testing.
 *
 * Creates:
 * - `.resist/keys/client-ca.pem` — Client CA certificate
 * - `.resist/keys/client-cert.pem` — Client certificate
 * - `.resist/keys/client-key.pem` — Client private key
 *
 * @param resistDir - Path to the `.resist/` directory.
 * @returns Result with paths to generated files, or error.
 */
function generateClientCertificates(
	resistDir: string,
): Result<ClientCertPaths> {
	const keysDir: string = `${resistDir}/keys`;

	// Generate client CA (separate from the server CA)
	// mkcert -client -cert-file .resist/keys/client-cert.pem \
	//        -key-file .resist/keys/client-key.pem \
	//        "client.localhost"
	const execResult = execSync(
		`mkcert -client -cert-file "${keysDir}/client-cert.pem" -key-file "${keysDir}/client-key.pem" "client.localhost"`,
	);
	if (!execResult.ok) return execResult;

	// The client CA is the mkcert root CA (same as server)
	// mkcert -CAROOT gives the CA directory
	const caRootResult = execSync('mkcert -CAROOT');
	if (!caRootResult.ok) return caRootResult;

	const caRoot: string = caRootResult.data.trim();
	const clientCaFile: string = `${caRoot}/rootCA.pem`;

	return okUnchecked({
		clientCaFile,
		clientCertFile: `${keysDir}/client-cert.pem`,
		clientKeyFile: `${keysDir}/client-key.pem`,
	});
}

/**
 * Paths to mkcert-generated client certificate files.
 */
type ClientCertPaths = {
	readonly clientCaFile: string;
	readonly clientCertFile: string;
	readonly clientKeyFile: string;
};
```

## 6. SSL Mode Behavior Summary

| Mode | Client to Caddy | Caddy to Upstream | Upstream Cert Validated? | Local Behavior |
|------|----------------|-------------------|--------------------------|----------------|
| `off` | HTTP | HTTP | N/A | No TLS listener, plain HTTP proxy |
| `flexible` | HTTPS | HTTP | N/A | TLS listener + HTTP upstream |
| `full` | HTTPS | HTTPS | No (`tls_insecure_skip_verify`) | TLS both legs, no upstream cert check |
| `strict` | HTTPS | HTTPS | Yes (mkcert CA trusted) | TLS both legs, upstream cert validated |
| `origin_pull` | HTTP or HTTPS (always upgrade) | HTTPS | Yes | Like strict + forced HTTPS upgrade |

## 7. Integration with `generateCaddyfile()`

The main Caddyfile generator in `edge/index.ts` calls `generateSslDirectives()` and assembles the fragments:

```typescript
// Inside generateCaddyfile() — per service block:
const sslResult: Result<SslDirectives> = generateSslDirectives(
	edgeConfig.ssl,
	certFile,
	keyFile,
	clientCaFile,
	hostname,
	httpsPort,
);
if (!sslResult.ok) return sslResult;

const ssl: SslDirectives = sslResult.data;

// Build site block
const siteLines: Array<string> = [];

// 1. Add redirect block (separate from main site)
if (ssl.redirect !== '') {
	siteLines.push(ssl.redirect);
	siteLines.push('');
}

// 2. Main site block
const scheme: string = edgeConfig.ssl.mode === 'off' ? 'http' : 'https';
siteLines.push(`${scheme}://${hostname}:${httpsPort} {`);

// 3. TLS block
if (ssl.tlsBlock !== '') {
	siteLines.push(ssl.tlsBlock);
}

// 4. SSL headers
if (ssl.headers !== '') {
	siteLines.push(ssl.headers);
}

// 5. Reverse proxy with transport
siteLines.push(`\treverse_proxy ${ssl.upstreamScheme}://localhost:${upstreamPort} {`);
if (ssl.transport !== '') {
	siteLines.push(ssl.transport);
}
if (ssl.authOriginPull !== '') {
	siteLines.push(ssl.authOriginPull);
}
siteLines.push('\t}');

siteLines.push('}');
```
