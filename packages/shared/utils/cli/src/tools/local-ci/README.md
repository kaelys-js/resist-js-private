# local-ci

Run CI workflows locally via [nektos/act](https://nektosact.com/) (GitHub Actions) or [gitlab-ci-local](https://github.com/firecow/gitlab-ci-local) (GitLab CI).

## Usage

```
pnpm tool local-ci <action> [flags]
```

## Actions

| Action | Description |
|--------|-------------|
| `run` | Run workflows/jobs locally (default) |
| `lint` | Validate workflow/CI config files |
| `list` | List available workflows and their jobs |
| `status` | Check prerequisites (Docker, runner tools) |

## Flags

| Flag | Short | Type | Description |
|------|-------|------|-------------|
| `--workflow` | `-w` | `string` | Path to a specific workflow file |
| `--job` | `-j` | `string` | Specific job ID to run |
| `--dry-run` | `-n` | `boolean` | Print command without executing |
| `--verbose` | `-v` | `boolean` | Show detailed output (passes `--verbose` to act) |
| `--json` | | `boolean` | Output results in JSON format for CI/scripting |
| `--filter` | `-f` | `string` | Filter workflows/jobs by substring match |

## How It Works

1. **Loads config** тАФ Reads `resist.config.ts` via `getConfig()` for provider, tool versions, CI settings
2. **Checks `tooling.ci.enabled`** тАФ Skips if disabled in config
3. **Detects provider** тАФ Routes to GitHub (act) or GitLab (gitlab-ci-local) based on `tooling.gitProvider.provider`
4. **Validates action** тАФ Validates against picklist: `run`, `lint`, `list`, `status`
5. **Checks prerequisites** тАФ Docker, runner tools (auto-installs via mise), config files
6. **Executes action** тАФ Dispatches to provider-specific handler with timing

### `local-ci run` (Default)

1. Checks prerequisites: Docker must be running, `act` must be installed
2. Auto-installs `act` via mise if missing (version from `config.versions.systemTools.act`)
3. Resolves workflow and job from flags (defaults to all workflows, all jobs)
4. Generates the `act` command with the configured runner image
5. Injects `--container-architecture linux/amd64` on Apple Silicon (ARM64)
6. Passes `--verbose` when `--verbose` flag is set
7. Passes `.secrets` file if it exists at the workspace root
8. Executes with timing тАФ displays elapsed seconds on completion
9. Propagates exit codes тАФ returns error on CI failure

### `local-ci lint`

1. Auto-installs `actionlint` via mise if missing (version from config)
2. Runs `actionlint` against all `.github/workflows/*.yml` files
3. Reports syntax errors, deprecated features, and best practice violations

### `local-ci list`

Lists all workflows in `.github/workflows/` and their jobs, parsed from YAML.

### `local-ci status`

Checks and reports the installation status of all prerequisites with formatted output:
- Docker status (running/not running)
- Tool versions (act, actionlint, or gitlab-ci-local)
- Config file status (.actrc or .gitlab-ci.yml)
- Workflow/job count

## Prerequisites

| Tool | Auto-Install | Version Source | Purpose |
|------|-------------|----------------|---------|
| Docker | **No** | тАФ | Container runtime |
| act | Yes (via mise) | `config.versions.systemTools.act` | GitHub Actions executor |
| actionlint | Yes (via mise) | `config.versions.systemTools.actionlint` | Workflow YAML linter |
| gitlab-ci-local | Yes (via mise) | `config.versions.systemTools['gitlab-ci-local']` | GitLab CI executor |
| `.actrc` | Generated by `pnpm tool sync` | тАФ | Runner image config |

## Configuration

In `resist.config.ts`:

```typescript
tooling: {
  ci: {
    enabled: true,        // Enable local CI (default: true)
    runnerSize: 'medium', // micro | medium | large
  },
  gitProvider: {
    provider: 'github',   // github | gitlab (default: github)
  },
  infisical: {
    siteUrl: 'http://localhost:8080',  // Infisical server URL
  },
},
versions: {
  systemTools: {
    act: '0.2.74',                  // Act version (managed by mise)
    actionlint: '1.7.11',           // Actionlint version (managed by mise)
    'gitlab-ci-local': '4.64.1',    // gitlab-ci-local version (managed by mise)
  },
},
```

### Runner Sizes

| Size | Image | Disk | Use Case |
|------|-------|------|----------|
| `micro` | `node:20-bookworm-slim` | ~200 MB | Node-only CI |
| `medium` | `ghcr.io/catthehacker/ubuntu:act-*` | ~500 MB | Standard tools |
| `large` | `ghcr.io/catthehacker/ubuntu:full-*` | ~17 GB | Full GitHub parity |

### Secrets

Create a `.secrets` file (gitignored) at the workspace root with `KEY=VALUE` pairs:

```
TURBO_TOKEN=xxx
TURBO_TEAM=xxx
```

Most build/test/lint jobs don't need secrets. Deploy steps fail gracefully without them.

If Infisical CLI is available, secrets are auto-generated from the development environment.

## Examples

```sh
pnpm tool local-ci                                    # Run all workflows
pnpm tool local-ci run                                # Explicit run action
pnpm tool local-ci run -w .github/workflows/ci.yml    # Run specific workflow
pnpm tool local-ci run -j lint                        # Run specific job
pnpm tool local-ci run --dry-run                      # Preview act command
pnpm tool local-ci run --verbose                      # Run with verbose output
pnpm tool local-ci lint                               # Lint workflow files
pnpm tool local-ci list                               # List workflows and jobs
pnpm tool local-ci list --json                        # List as JSON for scripting
pnpm tool local-ci status                             # Check prerequisites
pnpm tool local-ci run --filter ci                    # Run only workflows matching "ci"
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Completed successfully |
| `1` | Workflow failed or prerequisite missing |
| `2` | Invalid command usage or arguments |
| `3` | Unexpected fatal error |
| `130` | Interrupted (Ctrl+C) |
